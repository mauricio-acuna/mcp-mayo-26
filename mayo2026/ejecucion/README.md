## Reglas de oro de la ejecución

1. **C es prioridad estricta.** Si una semana hay que elegir, se trabaja en consultoría / outbound / engagement antes que en libro o OSS.
2. **A se trabaja en bloques.** 1 capítulo cada 2 semanas. Cero perfeccionismo: borrador publicable > obra maestra inacabada.
3. **E se mantiene en mínima viabilidad.** Hasta mes 6 es sólo el spec OWASP-MCP-Top-10 + repo placeholder. El scanner real no se construye hasta que la consultoría dé runway.
4. **Validación primero.** Si las 5 entrevistas dicen que el ángulo MCP-security en español **no** existe como dolor pagado, se reabren las opciones B o D antes de seguir.
