# Opción C — Consultoría / auditoría MCP-security

> Monetización inmediata del conocimiento ya producido. El libro pasa a ser portfolio.

## Tesis

En 2026 todas las empresas medianas y grandes están desplegando MCP servers internos sin saber lo que hacen en seguridad. El gap entre "tengo MCP" y "mi MCP no filtra datos al LLM" es enorme y es un servicio caro que no requiere construir producto.

## Activos a construir (no producto, sino entregables reutilizables)

- Plantilla de informe de auditoría MCP (markdown + checklist Excel).
- Modelo de amenazas STRIDE-MCP reutilizable.
- 30 controles prescriptivos con código de ejemplo.
- Caso de estudio anonimizado por engagement (alimenta blog y libro).

El **libro y el repo MediMind quedan como portfolio** de credibilidad — no como producto.

## Mercado realista

- **TAM ES + LatAm:** ~3.000 empresas con MCP en producción a fin de 2026.
- **Conversión realista:** 0.3% → 9 clientes/año.
- **Ingreso medio por cliente:** €15-25k.
- **Ingreso anual realista año 1:** €100-200k.

## Pros

- Time to revenue = semanas, no meses.
- Retroalimenta el contenido (cada engagement = capítulo / post).
- Alta tarifa horaria efectiva (€500-1.000/día).
- No requiere capital.

## Contras

- No escala sin contratar (negocio de servicios).
- Ingresos lumpy hasta tener pipeline estable.
- Vende tiempo del autor → no es activo vendible.

## Cuándo elegir esta opción

- El autor quiere ingresos rápido y no le molesta vender servicios.
- Tiene LinkedIn / red profesional ya activa o dispuesto a construirla rápido.
- Prefiere validar demanda con clientes pagando antes que construir producto.

## Combinación natural

C + A funcionan muy bien juntas: la consultoría paga el tiempo de escribir el libro, y el libro genera leads para la consultoría. Es el camino con mejor ratio riesgo/ingreso.
